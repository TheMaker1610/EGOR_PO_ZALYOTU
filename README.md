# Система оптимизации ТЭС

Программный комплекс для оценки эффективности и основных энергетических показателей энергооборудования тепловых электростанций.

---

## Требования

- Python 3.10+
- pip

---

## Установка

```bash
pip install -r requirements.txt
```

---

## Запуск

### Способ 1 — одной командой (macOS/Linux)

```bash
./start.sh
```

### Способ 2 — вручную (два терминала)

**Терминал 1 — сервер:**
```bash
python3 server.py
```

**Терминал 2 — клиент:**
```bash
python3 client.py
```

Сервер должен быть запущен **до** клиента. Дождитесь сообщения `Application startup complete.` в первом терминале, затем запускайте клиент.

---

## Вход в систему

При запуске клиента откроется окно входа.

| Логин | Пароль | Роль |
|-------|--------|------|
| `admin` | `Admin@123` | Администратор |
| `user1` | `User@12` | Пользователь |

> **Примечание:** при первом входе `user1` система потребует сменить пароль.

## Работа с программой

### Вкладка «Расчёт»

1. Введите параметры в левой панели:
   - **Нагрузка ТЭС** — суммарная электрическая нагрузка, МВт
   - **Кол-во блоков** — количество работающих энергоблоков
   - **Температура воздуха** — температура наружного воздуха, °C
   - **Влажность** — относительная влажность, %
   - **Скорость ветра** — м/с
   - **Направление ветра** — градусы (0–360)
2. При необходимости скорректируйте дополнительные параметры (номинальная мощность блока, КПД, коэффициент собственных нужд).
3. Нажмите **«Рассчитать»**.

В правой панели три вкладки с результатами:

**«Результаты»**

| Показатель | Описание |
|-----------|----------|
| Нагрузка на блок | МВт на каждый работающий блок |
| КПД блока | % с учётом нагрузки и внешних условий |
| КПД ТЭС брутто | % без учёта собственных нужд |
| Собственные нужды | МВт и % от нагрузки |
| КПД ТЭС нетто | % с учётом собственных нужд |
| Удельный расход топлива | г у.т./кВт·ч |

**«График (КПД от температуры)»** — зависимость КПД нетто и удельного расхода топлива от температуры воздуха в диапазоне −20…+40 °C.

**«Оптимизация числа блоков»** — таблица сравнения вариантов распределения нагрузки по числу блоков. Оптимальный вариант выделен зелёным цветом.

### Вкладка «История»

Показывает все ранее выполненные расчёты текущего пользователя: дата/время, параметры, результаты. Кнопка **«Обновить»** перезагружает список.

---

### Вкладка «Администрирование» (только для администратора)

#### Подвкладка «Пользователи»

| Действие | Как выполнить |
|----------|---------------|
| Просмотр пользователей | Список загружается автоматически |
| Создать пользователя | Кнопка **«Создать пользователя»** |
| Активировать / деактивировать | Выбрать строку → **«Вкл/Выкл аккаунт»** |
| Разблокировать | Выбрать строку → **«Разблокировать»** |
| Сбросить пароль | Выбрать строку → **«Сбросить пароль»** |

#### Подвкладка «Журнал событий (аудит)»

Отображает все зарегистрированные события системы: входы, выходы, блокировки, расчёты, изменения данных, API-запросы. Кнопка **«Обновить журнал»** перезагружает список с учётом текущего уровня детализации.

#### Подвкладка «Настройки журнала»

| Параметр | Описание |
|----------|----------|
| Макс. размер файла | Порог ротации лог-файла (МБ) |
| Кол-во архивных копий | Глубина хранения по числу файлов |
| Срок хранения | Глубина хранения по времени (дней) |
| Уровень детализации | ALL / DEBUG / INFO / WARNING / ERROR |
| URL удалённого сервера | Адрес для отправки логов (пусто = отключено) |

Кнопка **«Применить»** — применяет настройки немедленно без перезапуска сервера.
Кнопка **«Загрузить текущие настройки»** — считывает актуальные настройки с сервера.

Уровни детализации:

| Уровень | Что фиксируется |
|---------|----------------|
| ALL | Абсолютно все события |
| DEBUG | API-запросы |
| INFO | Входы, расчёты, смена паролей, управление пользователями |
| WARNING | Неверные пароли, неудачная смена пароля |
| ERROR | Блокировки аккаунтов |

#### Подвкладка «База данных»

Прямой просмотр содержимого SQLite-базы данных (только чтение):

| Вкладка | Таблица | Содержимое |
|---------|---------|------------|
| Users | `users` | пользователи, роли, статусы |
| Sessions | `sessions` | JWT-сессии, флаг отзыва |
| Calculations | `calculation_records` | история расчётов |
| Audit Logs | `audit_logs` | журнал событий (аудит) |

---

### Смена пароля

Кнопка **«Сменить пароль»** в синей панели в верхней части главного окна.

Требования к паролю:
- Пользователь: не менее **6** символов
- Администратор: не менее **7** символов
- Обязательно: заглавная буква (A-Z), строчная буква (a-z), цифра (0-9), спецсимвол (`~ @ % & * $ ^ ! #`)
- Пароль не должен содержать имя пользователя

### Выход из системы

Кнопка **«Выйти»** в синей панели → токен отзывается на сервере → возврат к окну входа.

---

## Политика безопасности

- При вводе пароля символы скрыты (`***`)
- После **3 неверных попыток** аккаунт блокируется на **15 минут**
- На эндпоинт входа установлен **rate limit 20 запросов/минуту** (защита от DDoS)
- Все события фиксируются в журнале аудита (БД) и в файле `logs/tes_audit.log`
- Каждый HTTP-запрос к API логируется с методом, путём, статусом и IP
- Журнал не содержит паролей и персональных данных
- Ротация лог-файла настраивается через интерфейс

| База данных | SQLite + SQLAlchemy ORM |

| Rate limiting | slowapi |
| GUI | PyQt5 |
| Графики | Matplotlib (встроен в PyQt5) |
| Логирование | Python logging + RotatingFileHandler |

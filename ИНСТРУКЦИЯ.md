# ШПАРГАЛКА — Что показывать на экзамене

---

## ЗАПУСК

```bash
python3 server.py   # терминал 1
python3 client.py   # терминал 2
```

Логин: `admin` / `Admin@123`

---

## ЧТО СПРОСИТ ПРЕПОДАВАТЕЛЬ → ЧТО ПОКАЗАТЬ

---

### «Покажи аутентификацию»

1. Открыто окно входа — **пароль скрыт звёздочками** (п. 3.2)
2. Введи неверный пароль 3 раза → сообщение **«Аккаунт заблокирован на 15 мин»** (п. 3.2)
3. Войди как `admin` / `Admin@123`
4. Войди как `user1` / `User@12` → сразу откроется **принудительная смена пароля** (п. 3.2)

---

### «Покажи требования к паролю»

При смене пароля введи слабый пароль (например `123`) → система покажет список ошибок:
- минимум 6 символов (пользователь) / 7 (администратор)
- нужна заглавная буква, строчная, цифра, спецсимвол (`~@%&*$^!#`)
- пароль не должен содержать имя пользователя

Кнопка «Сменить пароль» — в **верхней синей панели** главного окна.

---

### «Покажи расчёт»

Вкладка **«Расчёт»**:
1. Заполни поля (нагрузка, блоки, температура, влажность, ветер)
2. Нажми **«Рассчитать»**
3. Справа появятся результаты: КПД блока, КПД брутто, собственные нужды, КПД нетто, удельный расход топлива
4. Вкладка **«График»** — зависимость КПД от температуры −20…+40°C
5. Вкладка **«Оптимизация числа блоков»** — таблица + столбчатый график, оптимальный вариант выделен зелёным

---

### «Как учитываются внешние факторы?»

Объяснить: в полях ввода есть **температура, влажность, скорость и направление ветра** — каждый параметр влияет на поправочный коэффициент, который корректирует КПД. Поменяй температуру с +15 на +35 и пересчитай — КПД упадёт, расход топлива вырастет.

---

### «Покажи управление пользователями»

Вкладка **«Администрирование» → «Пользователи»** (только у admin):
- список всех пользователей, их роль, статус, попытки входа
- кнопка **«Создать пользователя»** → задать логин, пароль, роль
- кнопка **«Вкл/Выкл аккаунт»** → деактивировать/активировать
- кнопка **«Разблокировать»** → снять блокировку после 3 попыток
- кнопка **«Сбросить пароль»** → пользователь обязан сменить при следующем входе

---

### «Покажи журнал событий / аудит»

Вкладка **«Администрирование» → «Журнал событий»**:

Показать таблицу — в каждой строке видны все поля из п. 3.3.2:
- `#` — порядковый номер
- `Время` — дата и время события
- `Тип события` — LOGIN_SUCCESS, LOGIN_FAILED, CALCULATION и т.д.
- `Компонент` — объект доступа (auth, calc_engine)
- `Пользователь` — кто совершил действие
- `IP` — адрес
- `Служебные заголовки` — метод, путь, User-Agent
- `Детали` — дополнительный контекст

> Нажать «Обновить журнал» — новые события появятся сразу.

---

### «Что журнал НЕ содержит?» (п. 3.3.3)

Ответить: пароли (ни в открытом, ни в хэшированном виде), персональные данные, токены аутентификации. Показать столбец «Детали» — там только роль, номер попытки, нагрузка.

---

### «Покажи настройку глубины журнала» (п. 3.3.4)

Вкладка **«Администрирование» → «Настройки журнала»**:
- **Макс. размер файла** — порог ротации (объём)
- **Кол-во архивных копий** — глубина хранения
- **Срок хранения** — в днях
- **Уровень детализации** — ALL / DEBUG / INFO / WARNING / ERROR
- Нажать **«Применить»** — изменения вступают в силу немедленно

Показать переключение уровня: INFO → ALL → INFO.

---

### «Что значат уровни детализации?»

| Уровень | Что попадает в журнал |
|---|---|
| ALL | Абсолютно всё |
| DEBUG | Всё + технические запросы |
| INFO | Входы, расчёты, смена паролей (стандарт) |
| WARNING | Только неверные пароли |
| ERROR | Только блокировки аккаунтов |

---

### «Покажи ротацию журнала» (п. 3.3.6)

Объяснить: при достижении максимального размера файл автоматически архивируется, хранится заданное число копий. Настраивается в «Настройках журнала». Реализовано через `RotatingFileHandler`.

---

### «Покажи удалённую отправку логов» (п. 3.3.5)

В «Настройках журнала» поле **«URL удалённого сервера логов»** — вписать адрес сервера. Пустое поле = отключено.

---

### «Покажи защиту от перебора паролей / DDoS» (п. 4.1.3)

Объяснить: на эндпоинт входа установлен **rate limit 20 запросов/минуту** (slowapi). При превышении сервер возвращает ошибку 429. Плюс прикладная блокировка аккаунта после 3 попыток.

---

### «Покажи историю расчётов»

Вкладка **«История»** — таблица всех расчётов текущего пользователя: дата, нагрузка, блоки, температура, КПД нетто, расход топлива.

---

### «Покажи выход из системы»

Кнопка **«Выйти»** в верхней синей панели → токен отзывается на сервере (в журнале появляется `LOGOUT`) → возврат к окну входа.

---

## БЫСТРЫЙ СЦЕНАРИЙ ДЕМОНСТРАЦИИ

1. Запустить → окно входа → **ввести неверный пароль 3 раза** → показать блокировку
2. Войти как `admin` → показать **синюю панель** с кнопками
3. Вкладка «Расчёт» → заполнить → **«Рассчитать»** → показать результаты и график
4. Вкладка «Оптимизация числа блоков» → показать **зелёную строку**
5. «Администрирование» → «Пользователи» → **создать нового пользователя**
6. «Журнал событий» → показать **все колонки** записей
7. «Настройки журнала» → переключить уровень → **«Применить»**
8. «Выйти» → войти как новый пользователь → **принудительная смена пароля**
